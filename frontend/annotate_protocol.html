Annotation Protocol

CMF-Struct-v0.1

This document defines every field in the annotation schema. Read this before annotating. When in doubt, mark confidence 1–2 and add a note — do not guess.

Version history
CMF-Struct-v0.1 Initial release — surface-visible structural properties only.
Core principle

Annotate only what is visibly present in the rendered matrix entries. Do not normalize, simplify, or infer. If an entry is written as x/(3z), record it as a rational expression with a variable denominator — not as something that could be factored or simplified. Exact variable names matter here (record x, z, not a, b).

Tutorial cards

Card 1 — Matrix size

Definition: Matrix size = rows × columns. Count horizontal rows, then vertical columns.

2 rows, 2 columns → 2×2
3 rows, 1 column → 3×1

For the size signature, list each matrix left-to-right: 2x2|2x2|3x3

Card 2 — Visible variables

Definition: Any symbol that appears as a variable in the matrix entries. Record only what is literally shown.

Entries contain x+z and z² → visible variables: {x, z}
Entries contain n+1 and 2n → visible variables: {n}

Do not rename or normalize. If the matrix shows x, record x, not k.

Card 3 — Denominator present?

Definition: Mark yes if any matrix entry is written as a fraction (has a visible denominator).

x/3 → YES
(x+1)/(n+2) → YES
x + 1 → NO
x·(n+1) → NO

Card 4 — Variable in denominator?

Definition: Mark yes only if a variable (not just a number) appears in the denominator.

x/3 → NO (denominator is constant)
x/(3z) → YES (z is in denominator)
1/(n+1) → YES (n is in denominator)
(x+1)/4 → NO

This field can only be yes if denominator present is also yes.

Card 5 — Highest visible exponent

Definition: The largest explicit exponent appearing anywhere in the matrix. Look at all entries, all matrices.

Entries: x+z → highest exponent: 1
Entries: x²+z → highest exponent: 2
Entries: z⁴+xy → highest exponent: 4

Implicit degree-1 terms count as exponent 1. Constants have exponent 0.

Card 6 — Total degree (numerator / denominator)

Definition: Total degree of a monomial = sum of exponents of all variables.

x²y → total degree 3 (2+1)
xz⁴ → total degree 5 (1+4)
x+z → total degree 1 (each term is degree 1)
x²y/z³ → numerator degree 3, denominator degree 3

For a rational entry: record numerator and denominator separately. For a sum, record the maximum over all terms.

Card 7 — Entry type profile

Definition: Choose the most complex entry type present in the matrix.

constant_only — all entries are numbers (e.g. 0, 1, 3)
linear_polynomial — highest degree is 1 (e.g. x+1, 2n)
polynomial_degree_ge_2 — some entry has degree ≥ 2 (e.g. x², n(n+1))
rational_constant_denominator — fraction with constant denominator (e.g. x/3)
rational_variable_denominator — fraction with variable denominator (e.g. x/(n+1))

Card 8 — Zero entries

Definition: Mark yes if any matrix entry is visibly zero (0 or the empty expression).

Entry is 0 → YES
All entries are nonzero polynomials → NO

Card 9 — Symmetry impression

Definition: A rough visual impression only. Do not verify algebraically.

symmetric — M[i,j] = M[j,i] by inspection
possibly_symmetric — might be, hard to tell
not_symmetric — clearly asymmetric
unsure — cannot judge

Card 10 — Confidence score (1–5)

Use the 1–5 scale for every annotation:

1 = very unsure — high ambiguity, would need expert review
2 = somewhat unsure — plausible but not certain
3 = moderate — reasonable confidence
4 = fairly sure — only minor doubt
5 = very sure — no ambiguity

Prefer lower scores over forced guesses. A score of 2 with a note is more valuable than a score of 5 on a guess.

Card 11 — When you are unsure

Save as draft, add a note explaining the ambiguity, and give confidence 1–2. Do not submit annotations you are unsure about as submitted. Leave the ambiguous field blank rather than guessing.

Validation rules
  • ✓ visible_variable_count must equal the length of visible_variables
  • ✓ If has_variable_in_denominator = yes then has_denominator must also be yes
  • ✓ If has_denominator = no then highest_denominator_total_degree must be 0
  • ✓ Overall exponent / degree fields must be ≥ any per-matrix value
  • ✓ entry_type_profile = rational_variable_denominator requires has_variable_in_denominator = yes
  • ✓ entry_type_profile = rational_constant_denominator requires has_denominator = yes
  • ✓ If size signature is set, it must match per-matrix rows/cols
